# Osci Synth (Physics Synth)

Osci — это программный wavetable-синтезатор, написанный на C++. Проект ориентирован на производительность, визуализацию звука в реальном времени и drag-and-drop интерфейс управления модуляциями. 

В проекте не используются сторонние комплексные аудиофреймворки. Вывод звука реализован через собственный драйвер WASAPI (эксклюзивно для Windows), что обеспечивает минимальную задержку и прямой контроль над DSP-алгоритмами.

## Основные возможности

### Аудиодвижок
* Прямой вывод звука через кастомный WASAPI-драйвер.
* 128-голосная полифония.
* Поддержка классической полифонии и режима Mono Legato с настраиваемым параметром Glide (Portamento).

### Осцилляторы
* Два Wavetable-осциллятора (OSC A / OSC B) с поддержкой морфинга позиций (WT POS) и 3D-визуализации таблиц.
* Унисон: до 16 голосов на каждый осциллятор с настройками Detune и Stereo Blend.
* SUB-осциллятор: отдельный синусоидальный генератор саб-баса (смещение до -4 октав).
* Генератор шума с тремя алгоритмами: White Noise (Белый), Pink Noise (Розовый — фильтр Пола Келлета), Brown Noise (Коричневый — Leaky Integrator).

### Система модуляции
* Drag-and-Drop интерфейс для назначения модуляций от LFO или огибающих на параметры синтеза.
* Индикация глубины модуляции в реальном времени вокруг элементов управления.
* Вкладка MOD MATRIX для точной настройки связей (Amount от -100% до +100%) и их удаления.
* 3 ADSR-огибающие с интерактивными графиками (ENV 1 привязана к амплитуде, ENV 2 и 3 — свободные).
* 2 LFO с 4 формами волны, синхронизацией по частоте (Hz) или темпу (BPM).

### Фильтры и эффекты
* State Variable Filter: Low Pass, High Pass, Band Pass (12dB/oct) с визуализацией АЧХ и резонансом.
* Distortion: аналоговое насыщение на основе арктангенса (soft-clipping).
* Delay: стерео-дилей с настройками времени, фидбека и микса.
* Reverb: алгоритмический ревербератор Шредера.

### Визуализация и интерфейс
* Осциллограф с синхронизацией по переходу через ноль (Zero-crossing trigger).
* Анализатор спектра (FFT) на основе алгоритма Кули-Тьюки с окном Ханна.
* Встроенный редактор волн (вкладка DRAW) для отрисовки пользовательских форм сигнала.

### Управление
* Поддержка внешних USB MIDI-клавиатур через Windows MMAPI.
* Возможность игры на QWERTY-клавиатуре компьютера.
* Сохранение и загрузка пресетов в формате `.osci`.

## Стек технологий
* Язык: C++17
* Графика / GUI: Dear ImGui, OpenGL 3, GLFW
* Аудио API: Windows Audio Session API (WASAPI)
* MIDI API: Windows Multimedia API (WinMM)
* Сборка: CMake

## Требования для сборки
Проект предназначен исключительно для ОС Windows, так как жестко завязан на системные библиотеки `winmm`, `comdlg32` и `ole32`.

Для компиляции вам понадобятся:
* Windows SDK
* CMake (версии 3.14 или выше)
* Компилятор C++ с поддержкой стандарта C++17 (MSVC или MinGW/GCC)

Зависимости (GLFW и ImGui) загружаются и собираются автоматически средствами CMake (FetchContent).

## Инструкция по сборке

### Вариант 1: Через CMake (Командная строка)
1. Откройте терминал в корневой папке проекта.
2. Создайте директорию для сборки и перейдите в нее:

        mkdir build
        cd build

3. Сгенерируйте файлы сборки:

        cmake ..

4. Соберите проект:

        cmake --build . --config Release

5. Исполняемый файл `PhysicsSynthVisual.exe` будет находиться в директории `build/Release` (для MSVC) или прямо в `build` (для MinGW).

### Вариант 2: Использование MSYS2/MinGW (VS Code)
В проекте предусмотрен файл конфигурации `.vscode/tasks.json` для сборки с помощью `g++.exe` из среды MSYS2 (путь по умолчанию: `C:\msys64\ucrt64\bin\g++.exe`). 
Откройте проект в Visual Studio Code и запустите задачу сборки по умолчанию (Ctrl+Shift+B).

## Использование
1. Запустите `PhysicsSynthVisual.exe`.
2. Выберите устройство ввода в выпадающем списке `MIDI IN` на верхней панели (или используйте клавиатуру ПК).
3. Для назначения модуляции перейдите на вкладку `ENV` или `LFO`, зажмите кнопку `(+) DRAG MOD` и перетащите ее на любой энкодер (например, `Cutoff`).
4. Настройте параметры синтеза и сохраните результат, нажав кнопку `SAVE` на верхней панели.
